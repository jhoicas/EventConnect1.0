## 🚨 PRIORIDADES CRÍTICAS - FALTA IMPLEMENTAR

### **FASE INMEDIATA (CRÍTICO)**

1. **Sistema de Auditoría y Bitácora** (Módulo 5)
   - Tabla: `Auditoria` con campos (id, tabla_afectada, registro_id, usuario_id, accion, datos_anteriores, datos_nuevos, timestamp)
   - Endpoint: `GET /api/auditoria/historial/{entidad}/{id}` - Timeline visual
   - Triggers en BD para registrar cambios automáticamente
   - Servicio: `AuditoriaService`

2. **Gestión de Daños y Discrepancias** (Módulo 4)
   - Tabla: `Daño` (referencia a Reserva, Activo, Cliente, descripción, estado)
   - Endpoint: `POST /api/danos` - Crear ticket de daño
   - Endpoint: `GET /api/danos/{id}` - Ver daño
   - Actualizaciones a Hoja de Vida del Activo

3. **Alertas de Mantenimiento y Depreciación** (Módulo 2)
   - Integración con `MantenimientoController`
   - Sistema de notificaciones/alertas
   - Endpoint: `GET /api/activos/alertas` - Activos con mantenimiento pendiente

4. **Disponibilidad de Calendario** (Módulo 3)
   - Endpoint: `GET /api/reservas/disponibilidad` - Ver fechas disponibles por activo
   - Considerar bloques de reservas para evitar overlaps

## 🔗 FLUJO RECOMENDADO DE REGISTRO

```
1. POST /api/Auth/register-empresa
   → Registra empresa, devuelve empresa_Id

2. POST /api/Auth/register
   → Registra usuario Admin de la empresa
   → Usa empresa_Id del paso 1

3. POST /api/Auth/register-cliente
   → Cliente puede registrarse como persona o con empresa_Id
```
